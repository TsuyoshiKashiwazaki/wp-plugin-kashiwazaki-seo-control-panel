柏崎剛が公開する wp- 付きプラグイン/テーマの最新版を単一マニフェストから取得し、古いバージョンを検知して通知する作者専用コントロールパネル。アクセストークン不要。

== Description ==

WordPress 公式ディレクトリ未申請のプラグイン/テーマは標準の自動更新通知が効かず、古いまま放置されてハッキング被害に遭うリスクがあります。Kashiwazaki SEO ControlPanel は、作者が公開する単一のマニフェスト（全 wp- プラグイン/テーマの最新版情報をまとめた JSON）を取得し、ローカルに導入済みのバージョンが古い場合に管理画面とメールで警告します。マニフェストは 1 ファイルの取得だけで完結するため、アクセストークンは不要で、API レート制限の影響も受けません。

主な機能:

* マニフェストに基づく wp- 付きプラグイン/テーマの監視（本体を最上位に固定）
* インストール済みバージョンと最新版の差分照合（インストール名と公開名の表記揺れを吸収）
* セキュリティ更新の強調表示
* メール通知（セキュリティ更新は優先即時）
* wp-cron による定期チェック
* タブ式コントロールパネル（更新一覧・ニュース・設定・診断）
* WordPress ダッシュボードへの控えめな更新サマリ表示
* 本プラグイン自身の自己更新
* 作者ニュース最新6件の表示
* マニフェスト取得結果の Transient キャッシュ（取得失敗時も直前の良好データを保持）
* 外部通信の開示とニュース取得の無効化トグル

本バージョンは更新の「検知・通知・GitHub への誘導」までを行います。監視対象プラグイン/テーマの自動更新は行いません（本体自身の更新を除く）。

== Changelog ==

= 1.0.2 =
* 利用マニュアル（docs、スクリーンショット・図解付き）を追加。
* 更新一覧を「インストール済み / 未インストール」の 2 ブロックに分離し、「有効化」列（有効 / 無効）を新設。旧「状態」列は「更新のお知らせ」に改名。
* バージョン付き・zipball 形式などのディレクトリ名でインストールされた項目が「未インストール」と誤判定される問題を修正。
* 同じプラグインが複数ディレクトリに存在する場合の判定を決定的にした。
* 自己更新パッケージ URL の検証を強化（作者リポジトリ限定の多重防御）。
* マニフェスト URL を https のみ許可。
* 手動再取得の失敗時に成功と表示していた問題を修正（警告表示に変更）。
* 「除外する slug」の大文字小文字の揺れで除外が効かない問題を修正。
* ニュースの日付が不正な場合の 1970-01-01 表示を修正。
* ニュース取得失敗時のキャッシュ保持を短縮。
* 診断タブの不正な HTML 構造を修正、翻訳テンプレートを再生成。

= 1.0.1 =
* 更新一覧を再設計（統計タイル・更新ありを上部に表示・バージョン遷移表示・状態を色分け表示）。
* 更新の種類を「セキュリティ / バグ修正 / 機能」の 3 分類で判定し、複数該当時はすべて表示。
* ダッシュボードウィジェットに「セキュリティ / バグ修正 / 機能」の件数を種類別に表示。
* 管理画面ヘッダーとダッシュボードに、本プラグインの役割を示す説明文を追加。
* 作者ニュースの取得先を最適化（取得できない場合は RSS へフォールバック）。
* 管理画面 CSS の読み込みにファイル更新時刻を付与し、更新時にキャッシュが確実に切り替わるよう改善。
* 監視対象が多い環境で更新ステータスの保存に失敗することがある問題を修正。

= 1.0.0 =
* 初版リリース。
